# Python Reference Compare

End-to-end validation of the NVFP4 fused GEMM pipeline against Python reference output for layer 0's in_proj_qkv weight.

The test loads actual model weights (packed FP4, FP8 scales, global scale) from the safetensors file, saved as raw binary by a Python prep script (`/tmp/save_ref_binary.py`). It also loads a random BF16 input tensor and the Python-computed reference output (input @ dequant_weight^T).

The Rust test runs the fused `nvfp4_gemm_fused` kernel — which reads compressed FP4 weights directly, dequantizes in registers, and multiplies with BF16 activations without any intermediate bf16 buffer. Output: cosine similarity ≈ 0.99999, max_abs_err ≈ 0.031, mean_abs_err ≈ 0.004.

Reference data lives in `/tmp/` and is generated by `/tmp/save_ref_binary.py`: ref_weight_packed.bin [10240, 2560] uint8, ref_weight_scale.bin [10240, 320] uint8 (fp8_e4m3fn), ref_weight_global_scale.f32 (4-byte LE f32), ref_dequant_weight.bf16 [10240, 5120], ref_input.bf16 [1, 5120], ref_output.bf16 [1, 10240].
